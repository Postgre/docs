Connexions aux bases Oracle

 

Les drivers

Il existe 3 solutions pour se connecter à Oracle depuis Java :

  1. le pont jdbc :odbc
  2. le driver Oracle JDBC OCI (Oracle Call Interface)
  3. le driver Oracle JDBC Thin

 

La documentation sur l’utilisation des 2 drivers Oracle se trouve dans le chapitre « Application Development » de la documentation en ligne de Oracle.

Dans le cadre d’un serveur Web exécutant des servlets, vous ne devez en aucun cas utiliser les 2 premiers drivers. Ces drivers sont écris en C et tout « plantage » lors d’un appel aux fonctionnalités de ces drivers pourrait provoquer l’arrêt du serveur.

Configuration de IBM Visual Age for Java

Pour que VA puisse utiliser ce driver, il faut ajouter dans les ressources le chemin d’accès aux classes de ce driver :

Menu Fenêtre à Options à Ressources

Dans le champ « Chemin d’accès aux lasses pour l’espace de travail », tapez le chemin d’accès au fichier contenant le drivers.

Pour les JVM 1.1, ce fichier existe sur le serveur dia_dev dans le répertoire \dev\oracle\jdbc\classes111.zip.

Utilisation du driver dans Java

// charger le driver en mémoire

Class.forName("oracle.jdbc.driver.OracleDriver");

// obtenir une connexion

Connection conn = DriverManager.getConnection("jdbc:oracle:thin:@machine:port:sid", "userid", "password");

 

machine:          nom de la machine sur laquelle se trouve le moteur Oracle (machine de développement : dia_web)

port :                port du listener de la base (en général 1521)

sid :                 identifiant interne de la base à attaquer (sur la machine de développement : dev et hom)

 

Ces informations seront de préférences stockées dans un fichier « properties » (Cf java.util.properties). Elles ne doivent pas être codées « en dur » dans le source pour éviter les recompilations.
